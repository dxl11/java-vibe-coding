# 日志系统使用说明

## 一、日志架构

### 1.1 日志文件结构

```
logs/
├── application.log              # 应用日志（所有级别）
├── application-error.log        # 错误日志（ERROR级别）
├── application-business.log     # 业务日志（业务操作记录）
├── application-performance.log  # 性能日志（接口性能、慢查询等）
├── application-audit.log        # 审计日志（操作审计）
└── archive/                    # 归档日志
    ├── application-2024-01-13-0.log
    └── ...
```

### 1.2 日志级别

- **ERROR**: 系统错误、异常信息、业务失败
- **WARN**: 警告信息、潜在问题、降级处理
- **INFO**: 重要业务流程、关键节点、操作结果
- **DEBUG**: 调试信息、详细流程（生产环境关闭）
- **TRACE**: 最详细的跟踪信息（仅开发环境）

### 1.3 日志格式

#### 控制台格式
```
2024-01-13 10:30:45.123 [http-nio-8081-exec-1] INFO  com.vibe.user.service.impl.UserServiceImpl - 用户注册成功，用户ID: 1
```

#### 文件格式（包含 TraceId）
```
2024-01-13 10:30:45.123 [http-nio-8081-exec-1] INFO  [abc123def456] [1] [192.168.1.100] com.vibe.user.service.impl.UserServiceImpl - 用户注册成功，用户ID: 1
```

## 二、日志工具类使用

### 2.1 TraceIdUtils - 链路追踪

```java
// 生成 TraceId
String traceId = TraceIdUtils.generateTraceId();

// 设置 TraceId（通常在拦截器中自动设置）
TraceIdUtils.setTraceId(traceId);

// 获取当前 TraceId
String currentTraceId = TraceIdUtils.getTraceId();

// 清除 TraceId（通常在请求结束时自动清除）
TraceIdUtils.clearTraceId();
```

### 2.2 LogMaskUtils - 敏感信息脱敏

```java
// 脱敏手机号
String maskedPhone = LogMaskUtils.maskPhone("13812345678");  // 138****5678

// 脱敏身份证号
String maskedIdCard = LogMaskUtils.maskIdCard("110101199001011234");  // 110101********1234

// 脱敏银行卡号
String maskedBankCard = LogMaskUtils.maskBankCard("6222021234567890123");  // 6222****1234

// 脱敏密码
String maskedPassword = LogMaskUtils.maskPassword("123456");  // ******

// 脱敏邮箱
String maskedEmail = LogMaskUtils.maskEmail("username@example.com");  // u***@example.com

// 脱敏姓名
String maskedName = LogMaskUtils.maskName("张三");  // 张*

// 通用脱敏
String masked = LogMaskUtils.maskCommon("1234567890", 3, 4);  // 123****7890
```

### 2.3 LogUtils - 统一日志工具

```java
// 记录业务日志
LogUtils.businessLog("USER_REGISTER", "用户注册成功", userId, username);

// 记录审计日志
LogUtils.auditLog("USER_REGISTER", "USER", "1", "CREATE", "SUCCESS", "用户注册成功");

// 记录错误日志
LogUtils.errorLog(log, "用户注册失败", exception);

// 记录警告日志
LogUtils.warnLog(log, "库存不足", productId, stock);
```

### 2.4 PerformanceLogger - 性能日志

```java
// 记录接口性能
PerformanceLogger.logApiPerformance("POST", "/api/user/register", 150, true);

// 记录数据库查询性能
PerformanceLogger.logDbPerformance("SELECT * FROM user WHERE id = ?", 50);

// 记录RPC调用性能
PerformanceLogger.logRpcPerformance("inventory-service", "deductStock", 100, true);

// 记录消息队列性能
PerformanceLogger.logMqPerformance("order-topic", "order-create", "SEND", 20, true);
```

## 三、日志注解使用

### 3.1 @LogOperation - 操作审计注解

```java
@LogOperation(
    operation = "ORDER_CREATE",      // 操作类型
    resource = "ORDER",               // 资源类型
    action = "CREATE",                // 操作动作
    recordParams = true,              // 是否记录参数
    recordResult = true               // 是否记录结果
)
@GlobalTransactional(rollbackFor = Exception.class)
public OrderDTO createOrder(OrderCreateDTO createDTO) {
    // 业务逻辑
}
```

## 四、日志记录规范

### 4.1 Controller 层

```java
@RestController
@RequestMapping("/user")
public class UserController {
    
    @PostMapping("/register")
    public Result<UserDTO> register(@Validated @RequestBody UserRegisterDTO registerDTO) {
        // 请求参数和响应结果由 LoggingInterceptor 自动记录
        // 无需手动记录
        UserDTO userDTO = userService.register(registerDTO);
        return Result.success("注册成功", userDTO);
    }
}
```

### 4.2 Service 层

```java
@Service
public class UserServiceImpl implements UserService {
    
    @Override
    @LogOperation(operation = "USER_REGISTER", resource = "USER", action = "CREATE")
    public UserDTO register(UserRegisterDTO registerDTO) {
        // 记录业务日志
        LogUtils.businessLog("USER_REGISTER", "用户注册开始", 
                registerDTO.getUsername(), maskedPhone);
        
        // 业务逻辑...
        
        // 记录成功日志
        LogUtils.businessLog("USER_REGISTER", "用户注册成功", user.getId());
        
        return userDTO;
    }
}
```

### 4.3 异常处理

```java
try {
    // 业务逻辑
} catch (BusinessException e) {
    // 业务异常，记录警告日志
    LogUtils.warnLog(log, "业务异常", e.getMessage());
    throw e;
} catch (Exception e) {
    // 系统异常，记录错误日志
    LogUtils.errorLog(log, "系统异常", e);
    throw new BusinessException(500, "系统异常");
}
```

## 五、日志配置

### 5.1 环境变量

```bash
# 设置日志目录
export LOG_HOME=/var/log/vibe-ecommerce

# 设置应用名称
export SPRING_APPLICATION_NAME=user-service
```

### 5.2 application.yml 配置

```yaml
spring:
  profiles:
    active: dev

logging:
  config: classpath:logback-spring.xml
  level:
    root: INFO
    com.vibe: DEBUG  # 开发环境
```

### 5.3 日志文件管理

- **文件大小**: 单个文件最大 100MB
- **滚动策略**: 按日期和大小滚动
- **保留策略**: 
  - 应用日志：保留 30 天
  - 错误日志：保留 60 天
  - 业务日志：保留 90 天
  - 性能日志：保留 30 天
  - 审计日志：保留 365 天
- **总大小限制**: 各类型日志有总大小限制，超出后自动删除最旧的文件

## 六、最佳实践

### 6.1 敏感信息处理

**禁止**：
```java
log.info("用户密码: {}", password);  // ❌ 禁止
log.info("用户手机号: {}", phone);    // ❌ 禁止
```

**正确**：
```java
log.info("用户密码: {}", LogMaskUtils.maskPassword(password));  // ✅ 正确
log.info("用户手机号: {}", LogMaskUtils.maskPhone(phone));        // ✅ 正确
```

### 6.2 日志级别选择

- **ERROR**: 系统错误、异常、业务失败
- **WARN**: 警告、潜在问题、降级
- **INFO**: 业务流程、关键节点、操作结果
- **DEBUG**: 调试信息（生产环境关闭）

### 6.3 日志内容规范

- 包含必要的上下文信息（TraceId、用户ID、IP等）
- 使用结构化日志格式
- 避免记录过大的对象
- 敏感信息必须脱敏

### 6.4 性能日志

- 自动记录接口性能（通过 LoggingInterceptor）
- 自动记录慢查询（通过 MyBatis Plus 拦截器）
- 自动记录 RPC 调用性能（通过 Dubbo Filter）
- 自动记录消息队列性能（通过 AOP 切面）


## 九、注意事项

1. **生产环境**：DEBUG 级别日志会自动关闭，避免日志过多
2. **日志文件**：定期清理归档日志，避免磁盘空间不足
3. **敏感信息**：所有敏感信息必须脱敏处理
4. **性能影响**：日志记录不应影响业务性能，使用异步日志（Logback 默认异步）
5. **TraceId**：确保 TraceId 在服务间正确传递（通过 HTTP Header 或 Dubbo Attachment）
