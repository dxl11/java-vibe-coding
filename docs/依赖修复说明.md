# 依赖修复说明

## 修复的依赖问题

### 1. AOP 依赖缺失

**问题**：`common-core` 模块使用了 AspectJ 切面（`@Aspect`、`@Around`、`@Pointcut`），但缺少 AOP 依赖。

**修复**：在 `common-core/pom.xml` 中添加了 `spring-boot-starter-aop` 依赖。

```xml
<!-- AOP 支持（用于切面） -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>
```

**影响文件**：
- `common-core/src/main/java/com/vibe/common/core/log/aspect/MqLoggingAspect.java`
- `common-core/src/main/java/com/vibe/common/core/log/aspect/LogOperationAspect.java`

### 2. AspectJ 导入缺失

**问题**：`MqLoggingAspect.java` 中使用了 AspectJ 注解和类，但缺少导入语句。

**修复**：添加了必要的导入语句：
```java
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Pointcut;
```

### 3. 父 POM 版本管理缺失

**问题**：子模块使用了 Dubbo、RocketMQ、Spring Kafka，但父 POM 中没有版本管理。

**修复**：在父 `pom.xml` 的 `dependencyManagement` 中添加了版本管理：

```xml
<!-- Dubbo -->
<dependency>
    <groupId>org.apache.dubbo</groupId>
    <artifactId>dubbo-spring-boot-starter</artifactId>
    <version>${dubbo.version}</version>
</dependency>

<!-- RocketMQ Spring Boot Starter -->
<dependency>
    <groupId>org.apache.rocketmq</groupId>
    <artifactId>rocketmq-spring-boot-starter</artifactId>
    <version>${rocketmq-spring-boot-starter.version}</version>
</dependency>

<!-- Spring Kafka -->
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
    <version>${spring-kafka.version}</version>
</dependency>
```

## 解决依赖问题的方法

### 1. Maven 刷新

在 IDE 中执行：
- IntelliJ IDEA: `File` -> `Maven` -> `Reload Project`
- Eclipse: 右键项目 -> `Maven` -> `Update Project`

### 2. 清理并重新构建

```bash
mvn clean install -DskipTests
```

### 3. 检查依赖树

```bash
mvn dependency:tree
```

### 4. 检查依赖冲突

```bash
mvn dependency:analyze
```

## 常见问题

### Q1: 仍然无法导入某些类？

**A**: 检查以下几点：
1. 确认父 POM 中是否有版本管理
2. 确认子模块是否正确继承父 POM
3. 确认依赖的 scope 是否正确
4. 执行 `mvn clean install` 重新构建

### Q2: 版本冲突怎么办？

**A**: 
1. 使用 `mvn dependency:tree` 查看依赖树
2. 使用 `<exclusions>` 排除冲突的依赖
3. 在父 POM 的 `dependencyManagement` 中统一管理版本

### Q3: Spring Cloud Alibaba 依赖找不到？

**A**: 确认父 POM 中已添加：
```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-alibaba-dependencies</artifactId>
    <version>${spring-cloud-alibaba.version}</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>
```

## 验证修复

执行以下命令验证所有依赖是否正确：


如果编译成功，说明所有依赖问题已解决。
