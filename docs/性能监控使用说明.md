# 性能监控使用说明

## 一、功能概述

性能监控功能用于监控关键操作的执行时间，当执行时间超过预设阈值时，自动记录到性能日志中，便于发现性能问题。

## 二、核心特性

1. **注解驱动**：通过 `@MonitorPerformance` 注解标记需要监控的方法
2. **阈值配置**：支持通过注解参数或配置文件配置阈值
3. **异步记录**：默认异步记录，不影响业务性能
4. **自动记录**：超过阈值的方法自动记录到性能日志

## 三、使用方法

### 3.1 基本使用

在需要监控的方法上添加 `@MonitorPerformance` 注解：

```java
@MonitorPerformance(threshold = 2000, operation = "用户注册")
public UserDTO register(UserRegisterDTO registerDTO) {
    // 业务逻辑
}
```

### 3.2 注解参数说明

- `threshold`: 执行时间阈值（毫秒），默认 -1 表示使用配置文件默认值
- `operation`: 操作名称，用于日志记录，便于识别
- `async`: 是否异步记录，默认 true

### 3.3 配置示例

#### 使用注解配置阈值

```java
@MonitorPerformance(threshold = 2000, operation = "用户注册")
public UserDTO register(UserRegisterDTO registerDTO) {
    // 业务逻辑
}
```

#### 使用配置文件默认阈值

```java
@MonitorPerformance(operation = "订单创建")
public OrderDTO createOrder(OrderCreateDTO createDTO) {
    // 业务逻辑，使用配置文件中的默认阈值
}
```

### 3.4 配置文件

在 `application.yml` 中配置：

```yaml
vibe:
  monitor:
    performance:
      enabled: true              # 是否启用性能监控
      default-threshold: 2000    # 默认阈值（毫秒）
      record-all: false          # 是否记录所有方法（即使未超过阈值）
```

## 四、日志记录

### 4.1 日志位置

性能日志记录在：`logs/application-performance.log`

### 4.2 日志格式

```
METHOD|类名|方法名|操作名称|执行时间(ms)|阈值(ms)|级别|结果|TraceId
```

示例：
```
METHOD|UserServiceImpl|register|用户注册|2500|2000|SLOW|SUCCESS|abc123def456
```

### 4.3 日志级别

- **NORMAL**: 执行时间未超过阈值
- **SLOW**: 执行时间超过阈值

当执行时间超过阈值时，还会在应用日志中记录警告信息。

## 五、已应用监控的方法

### 5.1 用户服务

- `UserServiceImpl.register()` - 用户注册（阈值：2000ms）
- `UserServiceImpl.login()` - 用户登录（阈值：1000ms）

### 5.2 订单服务

- `OrderServiceImpl.createOrder()` - 订单创建（阈值：3000ms）

### 5.3 库存服务

- `InventoryServiceImpl.deductStock()` - 库存扣减（阈值：1000ms）

## 六、最佳实践

### 6.1 阈值设置

- **快速操作**（查询、简单计算）：500-1000ms
- **一般操作**（创建、更新）：1000-2000ms
- **复杂操作**（批量处理、分布式事务）：2000-5000ms

### 6.2 监控范围

建议监控以下关键操作：
- 用户注册、登录
- 订单创建、支付
- 库存扣减、回滚
- 支付处理
- 优惠券发放、使用

### 6.3 性能优化

当发现方法执行时间超过阈值时：
1. 查看性能日志，分析执行时间
2. 检查是否有慢查询
3. 检查是否有外部服务调用耗时过长
4. 优化业务逻辑或数据库查询

## 七、注意事项

1. **性能影响**：监控本身对性能影响很小（默认异步记录）
2. **日志量**：建议只监控关键方法，避免日志过多
3. **阈值设置**：根据实际业务场景合理设置阈值
4. **生产环境**：生产环境建议启用监控，便于发现性能问题

```java
    }
    
    @Override
    @MonitorPerformance(threshold = 1000, operation = "用户登录")
    public String login(UserLoginDTO loginDTO) {
        // 业务逻辑
        // 如果执行时间超过 1000ms，会自动记录到性能日志
    }
}
```
