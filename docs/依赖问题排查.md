# @Aspect 依赖问题排查指南

## 问题描述

如果遇到 `@Aspect`、`@Around`、`@Pointcut` 等注解无法导入的问题，请按以下步骤排查：

## 解决方案

### 1. 确认依赖已添加

`common-core/pom.xml` 中已包含：
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>
```

### 2. 刷新 Maven 项目

#### IntelliJ IDEA
1. 右键项目根目录
2. 选择 `Maven` -> `Reload Project`
3. 或者点击右侧 Maven 工具栏的刷新按钮

#### Eclipse
1. 右键项目
2. 选择 `Maven` -> `Update Project`
3. 勾选 `Force Update of Snapshots/Releases`

#### VS Code
1. 打开命令面板（Ctrl+Shift+P）
2. 输入 `Java: Clean Java Language Server Workspace`
3. 重启 VS Code

### 3. 清理并重新构建

```bash
# 在项目根目录执行
mvn clean install -DskipTests
```

### 4. 检查依赖是否正确下载

```bash
# 检查 common-core 模块的依赖
cd common-core
mvn dependency:tree | grep aop
```

应该能看到：
```
[INFO] +- org.springframework.boot:spring-boot-starter-aop:jar:2.7.18:compile
[INFO] |  +- org.springframework:spring-aop:jar:5.3.31:compile
[INFO] |  |  \- org.springframework:spring-beans:jar:5.3.31:compile
[INFO] |  \- org.aspectj:aspectjweaver:jar:1.9.7:compile
```

### 5. 验证 AspectJ 相关类

确认以下类可以正常导入：
- `org.aspectj.lang.annotation.Aspect`
- `org.aspectj.lang.annotation.Around`
- `org.aspectj.lang.annotation.Pointcut`
- `org.aspectj.lang.ProceedingJoinPoint`

### 6. 检查 IDE 设置

#### IntelliJ IDEA
1. `File` -> `Settings` -> `Build, Execution, Deployment` -> `Build Tools` -> `Maven`
2. 确认 Maven 配置正确
3. 确认 `Import Maven projects automatically` 已勾选

#### Eclipse
1. `Window` -> `Preferences` -> `Maven`
2. 确认 Maven 配置正确
3. 勾选 `Download repository index updates on startup`

### 7. 如果仍然无法导入

#### 方法1：手动刷新依赖
```bash
# 删除本地仓库中的缓存（谨慎操作）
# Windows: %USERPROFILE%\.m2\repository\org\springframework\boot\spring-boot-starter-aop
# Linux/Mac: ~/.m2/repository/org/springframework/boot/spring-boot-starter-aop

# 然后重新下载
mvn dependency:resolve
```

#### 方法2：检查 Maven 仓库配置
确认 `settings.xml` 中的仓库配置正确，可以访问 Maven 中央仓库。

#### 方法3：检查网络连接
确保可以访问 Maven 中央仓库或配置的镜像仓库。

### 8. 验证切面是否生效

创建一个测试类验证切面是否正常工作：

```java
@RestController
public class TestController {
    
    @MonitorPerformance(threshold = 1000, operation = "测试")
    @GetMapping("/test")
    public String test() {
        return "test";
    }
}
```

如果切面正常工作，访问该接口时会在日志中看到性能监控信息。

## 常见错误

### 错误1：Cannot resolve symbol 'Aspect'
**原因**：AOP 依赖未正确加载
**解决**：按照上述步骤刷新 Maven 项目

### 错误2：切面不生效
**原因**：
1. 切面类未被 Spring 扫描到
2. AOP 代理未启用
**解决**：
1. 确认切面类有 `@Component` 或 `@Aspect` 注解
2. 确认切面类在 Spring 扫描路径下
3. 确认 `spring-boot-starter-aop` 依赖已添加

### 错误3：编译通过但运行时切面不生效
**原因**：可能是 Spring AOP 代理配置问题
**解决**：确认使用 Spring AOP（基于代理），而不是 AspectJ 编译时织入

## 依赖说明

`spring-boot-starter-aop` 包含：
- `spring-aop`: Spring AOP 核心
- `aspectjweaver`: AspectJ 运行时支持
- `spring-core`: Spring 核心

这些依赖足以支持 `@Aspect`、`@Around`、`@Pointcut` 等注解的使用。
